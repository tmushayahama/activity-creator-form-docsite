Step 1. Create Model Metadata

  1. Enter Basic Activity Model in the title filed title
  2. Select Development as your model state
  3. Select an annotation group. In this case GO Central is selected
    Annotation groups are associated with curators in the users metadata file described above. By default, the first group associated with your entry in the metadata file is the group listed in the form. If you belong to multiple groups, you can select the appropriate group for your work from the drop-down list.
So far so good, let's Enter gene product

Enter gene product

Enter ABCA1 Hsap

By default, the form allows you to enter a single gene product. Start typing, choices will appear, and then select the gene product.

Tip: can type in the gene symbol, e.g. Wnt3a or the unique identifier or accession, e.g. UniProtKB:P56704. If necessary to narrow down the choices, type a space after the symbol, and enter the three letter code for the species (first letter from genus and two from species name, e.g. mmu for Mus musculus).  Each entry in the autocomplete will also show the associated unique database identifier or accession, so curators can confirm that they are selecting the appropriate entity for annotation.

Enter the molecular function, evidence, and reference

* Required Fields If:
  • you fill in the first field of any line, you need to add evidence and a reference.
  • the Molecular Function is known, enter the appropriate GO term, evidence code and reference.
  • the Molecular Function is unknown, but you are also making a Biological Process annotation, enter "molecular_function" and the same evidence and reference as the Biological Process annotation.
  • the Molecular Function is unknown, and there is no evidence for what the Molecular Function might be, enter "molecular_function" and the ND evidence code.
